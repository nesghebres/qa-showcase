## 1. Objective

The purpose of this test plan is to define the testing strategy and scope for validating the functionality, usability, and reliability of a web-based application.

This project demonstrates QA testing skills including test planning, test case design, and defect reporting.

---

## 2. Scope

### In Scope

- Login functionality
- Logout functionality
- Input validation
- Error handling
- Basic UI validation

### Out of Scope

- Performance testing
- Security penetration testing
- Load testing

---

## 3. Test Types

The following test types are used:

- Functional Testing
- UI Testing
- Negative Testing
- Regression Testing

---

## 4. Test Environment

- OS: Windows 11
- Browser: Chrome (latest version)
- Tool: Manual testing
- Repository: GitHub

---

## 5. Test Approach

Testing is performed manually by executing predefined test cases and documenting the results.

Each defect is recorded with:

- Steps to reproduce
- Expected result
- Actual result
- Severity

---

## 6. Entry Criteria

- Application is accessible
- Test cases are prepared

---

## 7. Exit Criteria

- All test cases executed
- All critical defects documented

---

## 8. Deliverables

- Test Plan
- Test Cases
- Bug Reports
